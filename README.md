# DocParser — Локальная система автозаполнения тендерных документов

## Описание

DocParser принимает входные документы (от заказчика и поставщика) + шаблоны целевых документов,
извлекает данные и формирует заполненные тендерные документы в формате DOCX.

## Стек

- Frontend: React + shadcn/ui + Vite
- Backend: Python + FastAPI + python-docx + Pydantic
- Deploy: Docker Compose

## Структура проекта

```
├── TZ.md                            ← техническое задание
├── rules.md                         ← правила для ИИ-агента
├── agent/                           ← навигация и поведение агента
├── agents/                          ← роли агентов (backend, frontend, qa, product)
├── sources-of-truth/                ← источники истины (бизнес, тех, архитектура)
├── phases/                          ← фазы разработки
├── hooks/                           ← руководство по хукам
├── Основной набор/                  ← входные документы (основной набор)
├── Контрольный набор с измененной формой/  ← входные документы (контрольный набор)
├── Шаблоны/                         ← пустые шаблоны целевых документов
├── Заполненные эталоны/             ← эталоны для сверки результата
└── (будет создано)
    ├── backend/                     ← Python + FastAPI
    ├── frontend/                    ← React + shadcn
    └── docker-compose.yml
```

## Запуск

```bash
docker compose up --build
```

Фронтенд: `http://localhost:5173`
API docs: `http://localhost:8000/docs`

## Порядок работы для агента

1. `rules.md` → главные правила
2. `agent/START.md` → навигация
3. `phases/phase-X/WHAT_TO_DO.md` → текущая задача
