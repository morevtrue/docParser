# Agent Pack — INVARIANTS (DocParser)

Критичные вещи, которые нельзя нарушать. Полный свод правил — в `rules.md`.

---

## Честность данных

- Система **никогда** не выдумывает данные.
- Если поле не найдено во входных документах — оно остаётся пустым / null.
- Каждое ненайденное поле попадает в warning отчёта.

## Устойчивость к изменениям форм

- Парсинг **не может** полагаться только на жёсткие индексы строк/ячеек.
- Семантический поиск по ключевым словам — основной механизм.
- Решение должно работать на основном И контрольном наборе без изменения кода.

## Разделение ответственности

- Бизнес-логика (парсинг, маппинг, генерация) — только на бэкенде.
- Frontend — только UI: загрузка, отображение, скачивание.
- Маппинг полей — конфигурация, отделённая от кода парсеров.

## Запрещено

- Использовать внешние LLM API.
- Хардкодить данные конкретных документов.
- Хардкодить плейсхолдеры шаблонов в коде генератора — вся логика "что куда" только в `FIELD_MAP`.
- Добавлять технологии без согласования.
