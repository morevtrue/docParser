# Agent Pack — TESTING (DocParser)

## Быстрые проверки

1. `docker compose up --build` — всё поднимается без ошибок.
2. `GET http://localhost:5173/api/health` → `{"status": "ok"}` (через Vite proxy)
3. Если задача в рамках фазы — открыть `phases/phase-X/TESTS.md`.

## Когда тесты обязательны

- Изменения в парсерах (проверить на основном наборе).
- Изменения в генераторе (сверить с эталонами).
- Изменения в API (проверить все коды ответов).
- Изменения в маппинге полей (проверить что ничего не сломалось).

## Принципы тестирования

- Тестируй через Swagger UI (`http://localhost:8000/docs`) или curl.
- Сверяй результат с эталонами из `Заполненные эталоны/`.
- Проверяй не только happy path, но и: пустые файлы, не-DOCX, частичные данные.
- Каждая фаза имеет свой чеклист в `TESTS.md`.
