# Phase 3 — Генерация выходных DOCX

**Статус:** ⏳ Не начата

---

## Цель фазы

Реализовать заполнение шаблонов DOCX извлечёнными данными. После этой фазы
`POST /api/process` принимает входные документы + шаблоны и возвращает ZIP
с заполненными документами + JSON-отчёт.

---

## Задачи

### Генератор (generator/filler.py)

1. Реализовать заполнение шаблонов:
   - Принимает Pydantic DTO (ExtractedData) + DOCX-шаблоны
   - Находит плейсхолдеры / пустые ячейки в шаблонах
   - Заполняет данными из DTO
   - Сохраняет форматирование оригинального шаблона

2. Поддержать три типа шаблонов:
   - Анкета участника
   - Заявка на участие в закупке
   - Предложение о цене договора

3. Реализовать маппинг "поле DTO → место в шаблоне":
   - Конфигурируемый: какое поле DTO куда вставляется в каждом шаблоне
   - Поиск по ключевым словам в шаблоне (не по индексам)

### Упаковка результатов

4. Собрать ZIP-архив:
   - 3 заполненных DOCX
   - report.json — отчёт об обработке

5. Обновить `POST /api/process`:
   - Полный pipeline: парсинг → маппинг → генерация → ZIP
   - Content-Type: application/zip
   - Отчёт включает: заполненные поля, пустые поля (warnings), ошибки

### Сверка с эталонами

6. Проверить результат на основном наборе:
   - Сравнить заполненные документы с эталонами из `Заполненные эталоны/`
   - Зафиксировать расхождения (если есть)

---

## Ограничения

- НЕ реализуем UI — тестируем через curl / Swagger.
- НЕ проверяем контрольный набор — это Phase 5.
- Сохранение форматирования шаблона — best effort (не все edge cases).
