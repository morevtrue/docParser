# Phase 4 — Frontend (UI)

**Статус:** ⏳ Не начата

---

## Цель фазы

Реализовать пользовательский интерфейс: загрузка файлов, отправка на бэкенд,
отображение отчёта, скачивание результата. После этой фазы пользователь может
работать с системой через браузер без curl/Swagger.

---

## Задачи

### Страница загрузки

1. Компонент загрузки файлов:
   - Drag & drop зона для DOCX-файлов
   - Кнопка "Выбрать файлы" (file picker) как альтернатива
   - Отображение списка загруженных файлов с возможностью удаления
   - Клиентская валидация: только .docx (по расширению и MIME-type)
   - Разделение на группы: "Документы от заказчика", "Документы поставщика", "Шаблоны"
     (или единая зона — на усмотрение, главное чтобы было понятно)

2. Кнопка "Обработать":
   - Отправляет файлы на `POST /api/process`
   - Показывает loading-состояние во время обработки

### Модальное окно ошибки

3. Если загружен не-DOCX файл:
   - Модальное окно: "Неправильный формат документа (Требуется DOCX)"
   - Кнопка "Загрузить другой документ"

### Отображение результата

4. После успешной обработки:
   - Отображение отчёта: заполненные поля, warnings, ошибки
   - Кнопка "Скачать результат" (ZIP-архив)
   - Кнопка "Обработать другие документы" (сброс формы)

### Состояния UI

5. Обработать все состояния:
   - Пустое (начальное) — форма загрузки
   - Loading — индикатор обработки
   - Success — отчёт + скачивание
   - Error — сообщение об ошибке от сервера

### Компоненты shadcn/ui

6. Использовать компоненты shadcn:
   - Button, Card, Dialog (модалка), Alert, Badge, Separator
   - Не создавать кастомные компоненты там, где есть shadcn-аналог

---

## Ограничения

- НЕ реализуем аутентификацию.
- НЕ реализуем историю обработок.
- НЕ реализуем предпросмотр документов.
