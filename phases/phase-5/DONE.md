### Контрольный набор

Система протестирована на контрольном наборе с изменённой формой без изменения core-логики.

Выявленные расхождения и исправления (расширение семантического поиска):

**Классификатор документов (`main.py`):**
- Добавлены ключевые слова: `"техническо-коммерческий запрос"`, `"технико-коммерческий запрос"`, `"досье поставщика"`, `"коммерческое предложение поставщика"`

**Парсер карточки поставщика (`supplier_card.py`):**
- Поддержка объединённого поля `"Краткое / полное наименование"` → разбивка на `short_name` / `full_name`
- Поддержка объединённого поля `"ИНН / КПП / ОГРН"` → разбивка по длине числа (ИНН=10/12, КПП=9, ОГРН=13/15)
- Поддержка объединённого поля `"Подписант / должность / основание"` → разбивка по запятой
- Поддержка `"Email для закупок"` как альтернативного ключа для email
- Логика: сначала проверяются специфичные объединённые ключи, затем стандартные

**Парсер коммерческих условий (`commercial_terms.py`):**
- Семантическое определение таблицы номенклатуры по заголовкам (не по индексу таблицы)
- Поддержка объединённого поля `"Поставщик / валюта"` → разбивка на `supplier_name` / `currency`
- Поддержка объединённого поля `"Поставка / гарантия"` → разбивка по ключевым словам
- Семантическое определение колонок таблицы номенклатуры по заголовкам

**Парсер запроса ТКП (`customer_request.py`):**
- Семантическое определение таблицы номенклатуры по заголовкам
- Поддержка объединённого поля `"Закупка / лот / код"` → разбивка на `purchase_number` / `lot` / `lot_code`
- Поиск полей в таблицах (не только в параграфах)
- Семантическое определение колонок таблицы номенклатуры по заголовкам

**Генератор (`filler.py`):**
- Семантическое определение колонок таблицы номенклатуры в шаблоне по заголовкам (функция `_detect_price_output_columns`)
