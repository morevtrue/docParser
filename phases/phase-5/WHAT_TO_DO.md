# Phase 5 — Интеграция, контрольный набор, документация

**Статус:** ✅ Завершена

---

## Цель фазы

Проверить систему на контрольном наборе с изменённой формой (без изменения кода),
подготовить документацию, сгенерировать итоговые документы, сделать скриншоты.
После этой фазы проект готов к сдаче.

---

## Задачи

### Контрольный набор

1. Прогнать систему на контрольном наборе (`Контрольный набор с измененной формой/`):
   - Загрузить документы с изменённой формой + шаблоны
   - Проверить, что парсинг работает без изменения кода
   - Сравнить результат с ожидаемым
   - Зафиксировать расхождения и warnings

2. Если парсинг не справляется с контрольным набором:
   - Доработать семантический поиск (расширить ключевые слова)
   - НЕ хардкодить под конкретный документ — улучшать общий механизм

### Генерация итоговых документов

3. Сгенерировать и сохранить:
   - Итоговые DOCX по основному набору → `output/основной/`
   - Итоговые DOCX по контрольному набору → `output/контрольный/`
   - Отчёты об обработке (JSON) для обоих наборов

### Документация

4. README.md:
   - Описание проекта
   - Инструкция запуска (`docker compose up --build`)
   - Как проверить (загрузить файлы, получить результат)
   - Структура проекта
   - Известные ограничения

5. Краткая документация по архитектуре:
   - Описание pipeline (парсинг → маппинг → генерация)
   - Как добавить новый тип документа
   - Как расширить маппинг полей

### Визуальная демонстрация

6. Подготовить 5–8 скриншотов:
   - Главная страница (форма загрузки)
   - Загруженные файлы в списке
   - Loading-состояние
   - Результат обработки (отчёт)
   - Скачанный ZIP с документами
   - Модальное окно ошибки (не-DOCX)
   - (опционально) Swagger UI с API

- НЕ добавляем новые фичи — только фиксим и документируем.
