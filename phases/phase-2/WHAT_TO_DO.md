# Phase 2 — Парсинг входных DOCX

**Статус:** ✅ Завершена

---

## Цель фазы

Реализовать извлечение данных из входных DOCX-документов. После этой фазы бэкенд умеет
принимать DOCX-файлы, парсить их и возвращать структурированные данные (Pydantic DTO).

---

## Задачи

### Pydantic DTO (models/schemas.py)

1. Определить модели данных:
   - `CustomerRequest` — данные из запроса ТКП (название закупки, номер, сроки, требования и т.д.)
   - `SupplierCard` — данные из карточки поставщика (название, ИНН, адрес, контакты и т.д.)
   - `CommercialTerms` — данные из коммерческих условий (позиции, цены, условия поставки и т.д.)
   - `ExtractedData` — объединённая модель всех извлечённых данных
   - `ParsingReport` — отчёт: какие поля найдены, какие нет, ошибки

### Парсеры (parser/)

2. Реализовать базовый парсер (`parser/base.py`):
   - Утилиты для работы с python-docx: извлечение текста из параграфов, таблиц
   - Семантический поиск по ключевым словам (нечёткое сопоставление)
   - Извлечение значения по ключу из таблицы (ключ в одной ячейке, значение в соседней)

3. Реализовать парсер запроса ТКП (`parser/customer_request.py`):
   - Извлечь: название закупки, номер, дата, сроки, требования к участникам
   - Проверить на основном наборе

4. Реализовать парсер карточки поставщика (`parser/supplier_card.py`):
   - Извлечь: наименование, ИНН, КПП, ОГРН, адрес, контакты, банковские реквизиты
   - Проверить на основном наборе

5. Реализовать парсер коммерческих условий (`parser/commercial_terms.py`):
   - Извлечь: позиции товаров/услуг, цены, условия поставки, сроки
   - Проверить на основном наборе

### Маппинг (mapping/field_map.py)

6. Создать конфигурацию маппинга полей:
   - Таблица: "поле DTO" → "ключевые слова для поиска" → "тип документа-источника"
   - Должна быть расширяемой без правки кода парсеров

### API-эндпоинт

7. Добавить `POST /api/process`:
   - Принимает multipart/form-data с DOCX-файлами
   - Валидация: только .docx файлы (иначе 400)
   - Классификация файлов: определить тип каждого документа (запрос ТКП / карточка / коммерческие условия / шаблон)
   - Парсинг каждого входного документа
   - Возвращает JSON с извлечёнными данными + отчёт (временно, пока нет генерации)

---

## Ограничения

- НЕ реализуем генерацию выходных документов — только парсинг и извлечение.
- НЕ реализуем UI загрузки — тестируем через curl / Swagger.
- Классификация документов может быть простой (по ключевым словам в содержимом).
