# BUSINESS_SPEC — Бизнес-спецификация DocParser

## Общая информация

**Тип продукта:** Internal tool / B2B utility
**Текущий статус:** MVP
**Целевая модель:** Локальное веб-приложение для автозаполнения тендерных документов.

**Краткое описание:**
DocParser — система, которая принимает входные документы (от заказчика и поставщика) и шаблоны,
извлекает из них данные и формирует заполненные тендерные документы в формате DOCX.

---

## Цель продукта

Создать систему, которая позволяет:
- Автоматически извлекать данные из входных DOCX-документов (запрос ТКП, карточка поставщика, коммерческие условия).
- Заполнять шаблоны целевых документов извлечёнными данными.
- Формировать отчёт об обработке (что заполнено, что не найдено, ошибки).

---

## Целевая аудитория

- Специалисты по тендерам / закупкам.
- Менеджеры, готовящие документацию для участия в закупочных процедурах.

| Роль         | Описание                          | Основные возможности                              |
|--------------|-----------------------------------|---------------------------------------------------|
| Пользователь | Специалист по тендерам            | Загрузка документов, получение заполненных DOCX    |

## Основные бизнес-сущности

### Входные документы
- Что это: DOCX-файлы от заказчика и поставщика.
- Типы: Запрос ТКП, Карточка поставщика, Коммерческие условия и цены.
- Принципы: поля могут менять порядок, формы могут незначительно отличаться.

### Шаблоны целевых документов
- Что это: пустые DOCX-шаблоны, которые нужно заполнить.
- Типы: Анкета участника, Заявка на участие в закупке, Предложение о цене договора.
- Принципы: пользователь загружает шаблоны вместе с входными документами.

### Выходные документы
- Что это: заполненные DOCX по шаблонам + JSON-отчёт об обработке.
- Принципы: если данные не найдены — поле пустое, не выдуманное.

### Отчёт об обработке
- Список успешно заполненных полей.
- Список полей, для которых данные не найдены (warnings).
- Ошибки парсинга, если были.

---

## Ключевые бизнес-правила

1. Система НЕ выдумывает данные. Если поле не найдено — оно пустое + warning.
2. Система должна работать на основном наборе документов И на контрольном наборе с изменённой формой без изменения кода.
3. Пользователь не готовит JSON вручную — только загружает DOCX.
4. Если загружен не DOCX — ошибка с понятным сообщением.
